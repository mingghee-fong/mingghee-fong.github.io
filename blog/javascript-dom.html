A Relationship Between JavaScript With HTML & CSS.

When we are building a house, there are 3 stages that we are considering.
1. The plan, foundation and structure.
2. The exterior and interior styling.
3. The features.

How does building a house related to HTML, CSS and JavaScript?

1. HTML plays the role of a builder.
It constructs the foundation and structure of the house.
Much like when you are building a website, the very first thing you need to build is the basic content that appears on your webpage.

2. CSS plays the role of a designer.
After your house is built, it's time to decide how you want your house to look on the outside and interior to suit your taste. Similarly, after you have built your HTML contents, you can use CSS to decorate your contents. For examples: you can change the style of your Fonts, decide your background colour and etc.

3. JavaScript on the other hand, is the feature of your house.
You have settled with the structure of your house, you've designed how you want your house to look like and finally it's time to decide the feature which will make your house into a home.
Think about JavaScript as furnitures and appliances. These are the features that are interactable, making your home more lively and interactive.
In short, JavaScript adds function to your webpage for user to interactive with.


Control Flow & Loops

Let's think about making a jam & toast. Such a simple idea but it still has a sequence of events before getting that perfect toast.


Getting Bread (Control Flow):
The control flow starts with the first step of making toast, which is getting a slice of bread.

Toaster Settings (Conditional Statements):
Next, you need to set the toaster to the desired level of toasting, which introduces a conditional decision. You adjust the toaster's settings based on your preference: light, medium, or dark toast.

Toasting (Loop Statements):
As you put the bread in the toaster and press the lever, a loop begins. The toaster repeatedly heats the bread until it reaches the selected level of toasting. This looping process ensures the bread is toasted evenly.

Checking (Conditional Statements):
Once the toaster finishes, you check the toast to see if it meets your desired level of toasting. If it's perfect, you proceed to the next step. If not, you might put it back in the toaster for a bit longer.

Butter and Jam (Control Flow):
If the toast is to your liking, you continue with the control flow. You follow the sequence of adding butter and jam (or any toppings you prefer) to the toast.

Eating (End of Control Flow):
Finally, you sit down and enjoy your toast, which marks the end of the control flow for this task.

In short, control flow is like a linear story telling. Control flow is the flow of the story, conditional is when you have to make a certain decision, loops is after you made decision then loop will bring you to the path you have chosen and eventually leading you to the end of the flow.


What Is A Document Object Model(DOM) & How To Interact With It?

Document Object Model(DOM) provides an interface for JavaScript to interact with HTML.

How so? Okay, DOM existence is purely for convenience.
After HTML structure and contents were built, going to back to the HTML to update in the future will be a pain as the content will grow bigger and bigger overtime.
Therefore, DOM is served as an interface/middleman for JavaScript to update and provide features at a later time.
It is faster and easier to use JavaScript to locate files and elements using codes and through DOM.


Differences Between Arrays and Objects & How To Access Them.

What is Arrays?

-Array is like a organized list of items that has a particular order.
-For example in my kitchen, I have a shelf full of seasoning and spices in containers.
-They are sorted by alphabetical order from left to right and that determines their location and we can easily count their indices.

How do we access data from Arrays?

-So i've mentioned something about "index" in the definition earlier.
-Normally, as a human when we are counting something, we would start from 1,2,3 first second third.
-But in JavaScript and some of the other programming languages, they would start counting from 0, then 1 2 3 and so on.

For example, if we want to pick up the 3rd container from the shelf, we would use this syntax from below:

let myContainer = ['almond','cumin','fennel','pepper'];

console.log(myContainer[2]),

Which will access the 3rd element in the array/shelf.
Meaning we will get "fennel as our result.
Remember that the index starts from 0.


What is Objects?

Object can be a jar labelled with let say, candy filled in it, it can also have quanity and expiry date as the values. It can be placed anywhere randomly in the cupboard. Unlike array, we will access object by its label/name.

How do we access data from Objects?

Accessing data from Object is like, going to the cupboard and look for the label.

For example, I have an onject/candy named "candyCherry". This cherry flavoured candy has about 31 pieces left in the jar and will be expiring on Janruary 2024. We want to look for the candy and also access the info in it,

let candyCherry = {
flavour: "cherry"
quantity: 31,
expiryDate: "Jan2024"
}

We will access it by using the code below, which will show us the entire values contained inside the object "candyCherry".

console.log(candyCherry);

To be more specific, we want to get the "quantity" data, we can use Dot Notation,

console.log(candyCherry.quantity);

and we get 31 in return!
